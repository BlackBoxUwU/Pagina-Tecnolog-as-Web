Protocolo HTTP

¿Qué es HTTP?

HTTP (Hypertext Transfer Protocol) es un protocolo de comunicación que permite la transferencia de información en la web. Es la base de cualquier intercambio de datos en la web y sigue un modelo de cliente-servidor, donde el cliente (navegador) solicita recursos y el servidor responde con los datos.

Métodos de HTTP
Método Descripción
GET Solicita un recurso sin modificarlo.
POST Envía datos al servidor para ser procesados.
PUT Actualiza un recurso o lo crea si no existe.
DELETE Elimina un recurso en el servidor.
PATCH Modifica parcialmente un recurso.
Códigos de estado HTTP
  • 100 - Informativos: Indican que la solicitud se ha recibido y el proceso continúa.
  • 200 - Éxito: La solicitud fue exitosa y el servidor devolvió la respuesta esperada.
  • 300 - Redirecciones: Indican que la solicitud debe ser redirigida a otra ubicación.
  • 400 - Errores del cliente: La solicitud tiene un error o falta autenticación.
  • 500 - Errores del servidor: Indican que el servidor encontró una situación inesperada.
URIs (Identificadores Uniformes de Recursos)

Los URIs (Uniform Resource Identifiers) son secuencias de caracteres que identifican recursos en la web. En HTTP, los URIs se utilizan para localizar y acceder a recursos específicos.

Clases de URIs:
  • URL (Uniform Resource Locator): Identifican recursos por su ubicación en la red. Ejemplo: https://www.ejemplo.com/index.html
  • URN (Uniform Resource Name): Identifican recursos por nombre en un espacio de nombres específico, independientemente de su ubicación. Ejemplo: urn:isbn:0451450523
Sintaxis Genérica de URI

esquema://autoridad/ruta?consulta#fragmento

  • Esquema: Indica el protocolo utilizado (http, https, ftp, etc.)
  • Autoridad: Generalmente contiene el nombre de dominio y, opcionalmente, el puerto
  • Ruta: Indica la ubicación específica del recurso en el servidor
  • Consulta: Contiene parámetros adicionales, precedidos por "?"
  • Fragmento: Referencia a una parte específica del recurso, precedido por "#"
